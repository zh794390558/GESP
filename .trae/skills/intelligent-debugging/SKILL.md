# 智能调试Skill
## Overview
本技能专注于为各类项目（Web应用、后端服务、脚本工具、客户端程序、接口服务等）提供全流程智能调试支持，
核心价值是在不破坏原有业务逻辑的前提下，快速定位代码报错、逻辑异常、运行故障的根本原因，
适配不同技术栈（Java、Python、Go、JavaScript、C/C++等）、不同运行环境与项目规模，
兼顾调试效率与系统稳定性，最小化改动、快速解决开发过程中的各类异常问题。

## When to Use
✅ 适用场景：
1. 代码编译/运行报错，包括语法错误、依赖冲突、类型异常、空指针、索引越界等；
2. 程序逻辑与预期不符，出现数据错误、死循环、卡死、结果异常、流程跳转错误；
3. 接口调用失败、超时、返回值异常、网络错误、鉴权失败、参数解析异常；
4. 程序崩溃、闪退、内存泄漏、CPU占用过高、文件句柄泄漏、资源未释放；
5. 日志排查，通过运行日志、错误堆栈定位系统故障与业务异常；
6. 环境配置问题，如依赖安装失败、端口占用、权限不足、环境变量配置错误。

❌ 不适用场景：
1. 从零编写全新业务代码（仅负责调试与修复问题，不开发新功能）；
2. 架构设计、代码重构、性能优化（仅聚焦问题定位与修复，不做结构优化）；
3. 非代码类问题（硬件故障、网络运维、服务器底层配置、数据库索引优化）；
4. 无报错信息、无代码、无日志的模糊问题（无法有效定位根因）；
5. 恶意代码、漏洞利用、非法程序与破解类调试需求。

## Workflow（必须按序执行）
1. 需求拆解：询问用户获取核心信息（技术栈、运行环境、报错堆栈、完整代码、日志、复现步骤）；
2. 问题定位：解析报错信息、代码逻辑与日志内容，锁定异常发生位置与异常类型；
3. 根因分析：深度分析问题成因，包括语法错误、逻辑错误、边界问题、空值、依赖与环境问题；
4. 修复方案：提供可直接复制的修复代码、配置修改、命令执行步骤，标注关键修复点；
5. 验证测试：指导用户复现与验证修复效果，确保问题彻底解决且不引入新Bug；
6. 优化建议：给出编码规范、异常处理、边界校验、日志增强等建议，降低同类问题复发概率。

## Examples
### 示例1：输入（Python 运行报错调试）
"Python 运行报错 IndexError: list index out of range，代码在遍历列表时崩溃，技术栈 Python3，帮忙调试修复。"

### 示例1：输出（核心调试方案）
#### 一、问题定位
1. 异常类型：数组索引越界
2. 故障位置：使用超出列表长度的下标访问元素
3. 根因：未做索引合法性判断，直接使用固定或动态索引访问列表

#### 二、修复方案
```python
# 错误代码
data = [10, 20, 30]
print(data[5])

# 修复代码
data = [10, 20, 30]
index = 5
if 0 <= index < len(data):
    print(data[index])
else:
    print(f"索引 {index} 不合法，列表长度为 {len(data)}")
